How to prepare for a release or other distribution

060425 Written EBJ
120719 Updated considerably

1.  Create a new release branch from the 'develop' branch:
    $ git checkout -b release/0.9.x

2.  Edit build.xml; change major_ver, full_ver strings as
    appropriate.

3.  Edit dist.make and change VERSTRING as appropriate.

4.  Edit DAVE/src/gov/nasa/daveml/dave/DAVE.java; change this.myVersion
    in DAVE constructor as appropriate (until we automate this)

5.  Edit overview.html; update 'revision history' section

6.  Update CHANGELOG.txt and README/README.txt summary of changes; run
    make in README directory

7.  Run 'ant distclean'

8.  Run 'ant doc'

9.  Look at docs, verify version number in title, etc.

10. 'ant test' & 'ant app-test'; ensure successful testing

11. Review TODO.txt list to see if any are resolved; update the list
    of changes

12. 'ant dist' to create .zip file

13. Send that file to another machine and verify 'ant doc', 'ant test'
    and 'ant app-test' work.

14. Final pass through README.txt to reflect changes

15. Add line to Release_log.txt about posting new version to branch
    website

16. A final 'git commit' in the release branch to record the
    updates.

17. Merge back into develop using 
    $ git checkout develop
    $ git merge --no-ff release/0.9.x

18. Merge back into master
    $ git checkout master
    $ git merge --no-ff release/0.9.x
    $ git tag -a 0.9.x

19. Run 'ant dist' to create a zip file

20. Put the new DAVEtools zip file on the DSCB Products website.

21. Copy the README/README.html into DAVEtools/index.html on the
    daveml.org website

22. Publish notices on daveml.org website; announce on sim-stds@nasa.gov
