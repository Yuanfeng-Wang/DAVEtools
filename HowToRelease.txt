How to prepare for a release or other distribution

060425 Written EBJ

1.  Edit build.xml; change major_ver, full_ver strings as
    appropriate.

1a. Edit dist.make and change VERSTRING as appropriate.

2.  Edit DAVE/src/gov/nasa/daveml/dave/DAVE.java; change this.myVersion
    in DAVE constructor as appropriate (until we automate this)

3.  Edit overview.html; update 'revision history' section

4.  'ant distclean'

5.  'ant doc'

6.  Look at docs, verify version number in title, etc.

7.  'ant test' & 'ant app-test'; ensure successful testing

8.  Update README.txt to fix version number; review bug list to see if
    any are resolved.

9.  Review and edit TODO if required.

10. 'ant distclean'

11. Check into repository ('git comment -a -m') with appropriate comment

12. 'git push central master' on development machine

13. Send that file to another machine and verify 'ant doc', 'ant test'
    and 'ant app-test' work.

=== stop here if alpha or beta version ===

15. Run 'ant dist' to create a new distribution in ../dist; add it to
    repository

16. Update Release_log.txt to reflect changes

17. A final 'git commit' will record the updates.

18. 'git tag DAVEtools_V009x' and 'git push central master'

19. Put the new DAVEtools zip file on the DSCB Products website.

20. Publish notices on daveml.org website; announce on sim-stds@nasa.gov
